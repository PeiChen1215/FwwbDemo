# 统一主键映射方案

## 目标

把所有来源表统一到一个标准字段 `student_id`，后续样本表固定使用 `学生-学期` 作为建模粒度。

## 统一标准

- 规范主键：`student_id`
- canonical 来源：`学生基本信息.xlsx` 的 `XH`
- 统一规范：
  - 所有主键先转成字符串
  - 去掉首尾空格
  - 保留原值列，同时新增标准化后的 `student_id`
  - 账号类字段统一转小写再匹配
  - 不对学号做数值化，避免丢失前导字符

## 映射优先级

1. 先做“直接可映射”的精确主键。
2. 再做“账号桥接”的高置信映射。
3. 最后才处理课程级或班级级数据回流到学生级。

## 第一组：直接精确映射

以下字段可以直接重命名或标准化后映射为 `student_id`：

- `XH`：`学生基本信息`、`学生成绩`、`学生选课信息`、`学籍异动`、`日常锻炼`、`体育课`、`体测数据`、`学生体能考核`、`考勤汇总`
- `XSBH`：`上网统计`、`奖学金获奖`、`社团活动`
- `SID`：`毕业去向`
- `USERNUM`：`跑步打卡`
- `IDSERTAL`：`门禁数据`
- `cardld`：`图书馆打卡记录`
- `KS_XH`：`四六级成绩`

这些表是第一版样本构建时最稳定的主数据来源。

## 第二组：条件精确映射

- `XHHGH`：来自 `学科竞赛`

处理规则：

- 只保留 `XHHGH` 在 `学生基本信息.XH` 中存在的记录
- 不在学生主表中的值，默认视为教师工号或未知主体
- 这部分记录标记为 `match_type = filtered_exact`

## 第三组：账号桥接映射

以下字段不是学号，而是平台账号：

- `LOGIN_NAME`：`学生签到记录`、`线上学习（综合表现）`、`课堂任务参与`
- `CREATER_LOGIN_NAME`：`学生作业提交记录`、`考试提交记录`、`讨论记录`
- `REPLY_LOGIN_NAME`：`讨论记录`

推荐桥接流程：

1. 先建立 `account_master`
   - 来源优先用 `线上学习（综合表现）`
   - 保留字段：`LOGIN_NAME`、`XM`、`DEPARTMENT_NAME`、`MAJOR_NAME`、`CLASS_NAME`

2. 再建立 `student_master`
   - canonical 来源：`学生基本信息`
   - 补充姓名和组织信息优先用 `学生体能考核` 与 `毕业去向`
   - 目标字段：`student_id`、`XM/REALNAME`、`YX/DEPARTMENT`、`ZY/MAJOR`、`BJ/CLASS`

3. 账号映射按以下顺序匹配
   - `姓名 + 院系 + 专业` 完全一致
   - `姓名 + 专业 + 班级` 完全一致
   - `姓名 + 院系` 一致且候选唯一

4. 无法唯一确定的账号先留在桥接表
   - 标记为 `match_type = unresolved`
   - 第一版 baseline 不把未解决账号表作为核心特征

## 第四组：课程级与班级级间接回流

以下表没有稳定学生主键，不能直接入模：

- `课程信息`
- `上课信息统计表`

处理方式：

1. 先统一课程主键
   - 标准课程键：`course_id`
   - 优先字段：`KCH`、`COURSE_ID`、`COURSE_CODE`、`JWCOURSE_ID`

2. 再统一教学班键
   - 标准教学班键：`class_course_id`
   - 优先字段：`JXBH`、`CLAZZ_ID`、`KXH`

3. 最后通过 `学生选课信息` 回流到学生
   - `学生选课信息` 是课程维表与学生行为表之间的关键桥接表

## 建议输出的桥接表字段

后续建议生成一张标准桥接表，例如 `student_key_bridge.csv`，字段至少包括：

- `source_table`
- `source_field`
- `source_value_raw`
- `source_value_norm`
- `student_id`
- `match_type`
- `match_confidence`
- `evidence_fields`

## 匹配置信度建议

- `exact`：直接学号字段精确映射
- `filtered_exact`：经过学生主表存在性过滤后的精确映射
- `bridge_high`：姓名、院系、专业三元组唯一匹配
- `bridge_medium`：姓名加班级或专业后唯一匹配
- `unresolved`：当前无法稳定映射

## 第一版建模建议只用这些表

如果你希望尽快跑通第一版 `学生-学期` baseline，优先使用以下高稳定表：

- `学生基本信息`
- `学生成绩`
- `学生选课信息`
- `学籍异动`
- `上网统计`
- `奖学金获奖`
- `本科生综合测评`
- `体测数据`

如果账号桥接完成，再把以下高价值表接进来：

- `学生签到记录`
- `线上学习（综合表现）`
- `课堂任务参与`
- `学生作业提交记录`
- `考试提交记录`
- `讨论记录`

## 当前结论

- `XH` 应作为全项目唯一 canonical student_id
- 直接学号类表可以立即进入第一版样本构建
- 账号类表价值很高，但必须先做桥接表
- `上课信息统计表` 和 `课程信息` 更适合作为课程侧环境特征，而不是直接学生特征
- `跑步打卡` 需要先确认是否存在 Excel 截断，再决定是否纳入主流程
