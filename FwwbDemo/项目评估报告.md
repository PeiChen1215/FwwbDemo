## 二、技术方案评价

### 2.1 整体架构

```
原始数据 → ETL处理 → 学期粒度宽表 → SSL预训练 → 嵌入提取 → 密度聚类 → 监督学习
```

### 2.2 核心技术点评分

| 技术模块 | 实现方案 | 评分 | 评价 |
|----------|----------|:--:|------|
| **自监督学习** | DAE(去噪自编码器) + Transformer | ⭐⭐⭐⭐ | 有效利用无标签数据，提取64维高质量嵌入 |
| **对比学习** | NNCLR + Memory Bank | ⭐⭐⭐⭐ | 挖掘学生间相似性，增强表示学习 |
| **特征编码** | FT-Tokenizer + Transformer Encoder | ⭐⭐⭐⭐ | 适合表格数据，比MLP更具表达力 |
| **聚类算法** | HDBSCAN密度聚类 | ⭐⭐⭐⭐ | 自动发现异常值，无需预设K值 |
| **分类器** | Logistic Regression | ⭐⭐⭐ | 过于简单，未尝试XGBoost等SOTA模型 |

### 2.3 技术优势

1. **解决标签稀缺问题**: 通过SSL预训练，充分利用19,758条无标签数据
2. **捕捉高阶关系**: Transformer的自注意力机制建模特征间复杂交互
3. **自动异常检测**: HDBSCAN识别出13名高风险异常学生（Outliers）
4. **可解释性强**: PCA可视化清晰展示4类学生群体的分离度

## 四、存在问题诊断

### 4.1 关键问题列表

#### 🔴 P0级（严重影响实用性）

**1. 类别极度不平衡未处理**
- **现状**: 正样本仅0.94%，模型倾向预测全0
- **后果**: AUC虚高(0.8373)，但F1≈0.04，实际无法识别风险学生
- **证据**: 测试集3,519个样本中，模型几乎预测全部为0类

**2. 最优模型过于简单**
- **现状**: LogisticRegression胜出，未使用XGBoost/LightGBM
- **后果**: 可能欠拟合，未充分挖掘特征非线性关系
- **风险**: 在复杂场景下泛化能力不足

#### 🟡 P1级（影响性能上限）

**3. 特征工程粗糙**
- **缺失RFM特征**: 未提取行为的Recency/Frequency/Monetary特征
- **缺失时序特征**: 成绩趋势、行为波动性、学习规律性是重要指标
- **缺失交叉特征**: 如"学习效率=成绩/学习时长"、"作息规律性"等
- **数据源未充分利用**: 图书馆(覆盖率19%)、跑步打卡(覆盖率21%)未接入

**4. 聚类结果利用率低**
- 发现4类模式，但未用于监督学习
- 未尝试"分簇建模"（不同簇用不同分类器）
- 未将簇标签作为特征输入模型

#### 🟢 P2级（影响答辩表现）

**5. 缺乏充分对比实验**
- 无消融实验（Ablation Study）
- 未对比不同嵌入维度（32/64/128）
- 未对比传统方法（K-Means+LR vs SSL+LR）

**6. 可视化不足**
- 缺乏特征重要性分析图
- 缺乏学习曲线（Loss Curve）
- 缺乏混淆矩阵和ROC曲线

## 五、冲刺国一改进方案

### 5.1 短期优化（1-3天，预期AUC提升至0.88+）

#### 方案A: 处理类别不平衡

**方法1: SMOTE过采样**
```python
from imblearn.over_sampling import SMOTE
smote = SMOTE(random_state=42, k_neighbors=5)
X_resampled, y_resampled = smote.fit_resample(X_train, y_train)
```

**方法2: 调整分类阈值**
```python
# 降低阈值，提高Recall
y_pred = (model.predict_proba(X_test)[:, 1] > 0.3).astype(int)
```

**方法3: 类别权重**
```python
model = LogisticRegression(class_weight={0: 1, 1: 100})  # 增大正样本权重
```

**预期提升**: AUC +0.02~0.03，F1-Score大幅提升

#### 方案B: 引入XGBoost/LightGBM

```python
import xgboost as xgb

# XGBoost with scale_pos_weight处理不平衡
model = xgb.XGBClassifier(
    n_estimators=200,
    max_depth=6,
    learning_rate=0.05,
    scale_pos_weight=100,  # 负样本数/正样本数
    eval_metric='auc'
)
```

**预期提升**: AUC +0.02~0.05

#### 方案C: 特征融合

```python
# 原始特征(15维) + SSL嵌入(64维) + 聚类标签(1维) = 80维
X_combined = np.hstack([X_raw, X_emb, cluster_labels.reshape(-1, 1)])
```

**预期提升**: AUC +0.01~0.02

### 5.2 中期优化（3-5天，预期AUC 0.90+）

#### 方案D: 深度特征工程

**1. RFM特征提取**
- Recency: 最近图书馆/上网距今天数
- Frequency: 每周平均学习次数
- Monetary: 总学习时长

**2. 时序特征**
- 成绩趋势（线性回归斜率）
- 成绩波动性（标准差）
- 学习投入变化率

**3. 交叉特征**
```python
df['study_efficiency'] = df['avg_score'] / (df['internet_hours_sum'] + 1)
df['health_index'] = df['physical_test_score'] * df['sleep_regularity']
df['social_learning_ratio'] = df['forum_posts'] / df['library_hours']
```

**预期提升**: AUC +0.03~0.05

#### 方案E: 接入更多数据源

**图书馆数据挖掘**:
- 入馆频率、偏好时段（晨型/夜型人）
- 学习时长分布、规律性
- 与成绩相关性分析

**跑步打卡数据**:
- 运动频率、坚持天数
- 体能与健康指标

**预期提升**: AUC +0.02~0.03

### 5.3 长期优化（5-7天，冲刺顶级）

#### 方案F: 因果推断

使用DoWhy库分析因果关系：
```python
import dowhy

# 分析"增加图书馆时长"对"成绩提升"的因果效应
causal_model = dowhy.CausalModel(
    data=df,
    treatment='library_hours',
    outcome='exam_score',
    common_causes=['gender', 'major', 'previous_score']
)
```

**价值**: 提供"什么干预对什么学生有效"的洞察，增强可解释性

#### 方案G: 图神经网络(GNN)

构建学生相似性图：
- 节点: 学生
- 边: 行为相似度
- 消息传递: 相似学生互相影响

**价值**: 捕捉社交影响和学习伙伴效应
